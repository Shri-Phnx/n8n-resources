# n8n Credentials Guide — Part 1: API Keys & OAuth2\n\n> Author: Shrinivas Ramaprasad | April 2026 | See Part 2 for Telegram, Slack, GitHub, SMTP, DB credentials.\n\n---\n\n## How Credentials Work\n\nCredentials are encrypted authentication records. Encrypted with `N8N_ENCRYPTION_KEY`. Decrypted only in memory during execution. Never visible in logs.\n\n**Access:** Settings → Credentials → + Add Credential  \n**Naming:** `ServiceName - Purpose - Environment` (e.g., `OpenAI - CareerForge Prod`)\n\n---\n\n## Type 1 — API Key\n\n**Used by:** OpenAI, Anthropic, Groq, NVIDIA NIM, Hugging Face, most REST APIs.\n\n| Field | Required | Sample Value | What It Does |\n|---|---|---|---|\n| **Credential Name** | Yes | `OpenAI - CareerForge Prod` | Display name only |\n| **API Key** | Yes | `sk-proj-abc123...` | Secret key sent with every API request |\n\n### OpenAI\n```\n1. platform.openai.com → Profile → API keys → + Create new secret key\n   Name: n8n-automation | Copy key (starts with sk-proj-)\n   ⚠️ SHOWN ONLY ONCE\n2. n8n: Settings → Credentials → + Add → OpenAI\n   API Key: paste → Save → ✅ Connected\n```\n\n### Anthropic (Claude)\n```\n1. console.anthropic.com → Settings → API Keys → + Create Key\n   Name: n8n | Copy (starts with sk-ant-api03-)\n2. n8n: Credentials → + Add → Anthropic\n   API Key: sk-ant-api03-... → Save\n```\n\n### Groq\n```\n1. console.groq.com → API Keys → Create API Key | Copy (starts with gsk_)\n2. n8n: Credentials → + Add → Groq\n   API Key: gsk_... → Save\n```\n\n### NVIDIA NIM\n```\n1. build.nvidia.com → Profile → Generate API Key | Copy (starts with nvapi-)\n2. n8n: Use via HTTP Request → Header Auth credential\n   Header Name: Authorization | Header Value: Bearer nvapi-your-key\n```\n\n### HuggingFace\n```\n1. huggingface.co → Settings → Access Tokens → New Token (Read role)\n2. n8n: Credentials → + Add → HuggingFace\n   API Key: hf_your-token → Save\n```\n\n**✅ API Key Best Practices:**\n- Separate keys per application AND per environment\n- Save to password manager IMMEDIATELY — shown only once\n- Rotate every 90 days for production\n- Use scoped/restricted keys where available (OpenAI: Project keys)\n\n**⚠️ Common Mistakes:**\n| Mistake | Effect | Fix |\n|---|---|---|\n| Sharing key across environments | Revoking dev breaks prod | Separate keys per env |\n| Not saving key | Must regenerate | Password manager first |\n| Spaces when pasting | 401 Unauthorized | Trim whitespace |\n| Hardcoding in URL/expression | Exposed in exports | Always use credential reference |\n\n---\n\n## Type 2 — Google OAuth2\n\n### Option A: Quick Sign-In (Personal)\n```\n1. n8n: Settings → Credentials → + Add → Gmail OAuth2 API\n   (same credential for Sheets, Drive, Calendar, Docs)\n2. Credential Name: Google - Shrinivas Personal\n3. Click \"Sign in with Google\" → select account → Allow\n4. Status: ✅ Connected (token auto-refreshes)\n```\n\n### Option B: Cloud Console (Workspace / Custom App)\n\n**Step 1 — Enable APIs:**\n```\n1. console.cloud.google.com → New Project: n8n-automation\n2. APIs & Services → Library → Enable:\n   Gmail API, Google Sheets API, Google Drive API,\n   Google Calendar API, Google Docs API,\n   YouTube Data API v3 (if needed)\n```\n\n**Step 2 — OAuth Consent Screen:**\n```\n1. APIs & Services → OAuth consent screen\n2. User Type: External (personal) or Internal (Workspace)\n3. App name: n8n Automation | Support email: your@gmail.com\n4. Test users: add your Gmail address\n5. Save and Continue\n```\n\n**Step 3 — Create OAuth Client ID:**\n```\n1. Credentials → + Create Credentials → OAuth client ID\n2. Application type: Web application\n3. Name: n8n Client\n4. Authorized redirect URIs (EXACT — no trailing slash):\n   https://your-n8n.com/rest/oauth2-credential/callback\n   OR (local): http://localhost:5678/rest/oauth2-credential/callback\n5. Create → Copy Client ID and Client Secret\n```\n\n**Step 4 — Configure in n8n:**\n\n| Field | Sample Value | Where to Get |\n|---|---|---|\n| **Credential Name** | `Google OAuth2 - Work` | You choose |\n| **Client ID** | `123456789-abc.apps.googleusercontent.com` | Google Cloud → Credentials |\n| **Client Secret** | `GOCSPX-abc123...` | Google Cloud → Credentials |\n| **Scope** | Auto-filled | Leave default |\n\n```\n→ Click Sign in with Google → authorize → ✅ Connected\n```\n\n**⚠️ Common OAuth2 Mistakes:**\n| Mistake | Error | Fix |\n|---|---|---|\n| Trailing slash in redirect URI | `redirect_uri_mismatch` | Remove `/` at end |\n| Using http:// in production | OAuth rejected | Must use https:// |\n| API not enabled | `403 accessNotConfigured` | Enable API in Cloud Console Library |\n| Token expired (6+ months) | `401 invalid_grant` | Re-authenticate in Settings → Credentials |\n| Wrong account | Data from wrong mailbox | Delete credential → re-authorize |\n| User not in test list | `403 access_blocked` | Add email to OAuth consent screen test users |\n\n---\n\n## Type 3 — Google Service Account\n\n**Use when:** Automating Google Workspace without human OAuth popup.\n\n```\nStep 1:\n1. console.cloud.google.com → IAM & Admin → Service Accounts\n2. + Create Service Account\n   Name: n8n-automation | Role: Editor (or specific roles)\n3. Keys tab → Add Key → JSON → Create → Download JSON file\n\nStep 2 — Domain-Wide Delegation:\n1. admin.google.com → Security → API Controls → Domain-wide Delegation\n2. Add new:\n   Client ID: (from JSON → \"client_id\" field)\n   Scopes: https://mail.google.com/,\n           https://www.googleapis.com/auth/spreadsheets,\n           https://www.googleapis.com/auth/drive,\n           https://www.googleapis.com/auth/calendar\n\nStep 3 — Configure in n8n:\nSettings → Credentials → + Add → Google Service Account\n```\n\n| Field | Sample | Source |\n|---|---|---|\n| **Service Account Email** | `n8n-auto@project.iam.gserviceaccount.com` | JSON → `client_email` |\n| **Private Key** | `-----BEGIN PRIVATE KEY-----\\nMIIE...` | JSON → `private_key` (full block) |\n| **Impersonate User** | `shrinivas@yourcompany.com` | Workspace user to act as |\n
