AI is optional / fallback to open-source APIs.\n\n---\n\n## How to Use These Templates\n\nEach template includes:\n1. The workflow node structure (what connects to what)\n2. Key node configurations\n3. Where to add AI (optional — always with free/open-source fallback)\n4. The JSON template you can import directly into n8n\n\n**To import a template:**\n```\nn8n → top-right menu (☰) → Import from File\n→ paste the JSON → Import\n→ Add your credentials → Activate\n```\n\n---\n\n## Template 1 — IT Support Telegram Bot\n\n### What it does\nUsers send messages to a Telegram bot. The bot creates tickets, notifies the team on Slack, logs to Google Sheets, and sends confirmation back.\n\n### Node Structure\n```\nTelegram Trigger (receives /ticket command)\n    ↓\nIf: message starts with /ticket?\n    ├── YES:\n    │   Edit Fields (build ticket object)\n    │       ticketId: TICK-{{ $now.toMillis() }}\n    │       user: {{ $json.message.from.first_name }}\n    │       issue: {{ $json.message.text.replace('/ticket ','') }}\n    │       chatId: {{ $json.message.chat.id }}\n    │       status: Open\n    │       createdAt: {{ $now.toISO() }}\n    │   ↓\n    │   [PARALLEL BRANCHES]\n    │   ├── Google Sheets: Append Row (log ticket)\n    │   ├── Slack: Send Message to #it-support\n    │   └── Telegram: Reply to user\n    └── NO:\n        Telegram: Send \"Use /ticket <describe your issue>\"\n```\n\n### Google Sheets Setup\n```\nSpreadsheet: IT Support Tracker\nSheet: Tickets\nColumns: Ticket ID | User | Issue | Status | Created At | Resolved At\n\nAppend Row mapping:\n  Ticket ID:   {{ $json.ticketId }}\n  User:        {{ $json.user }}\n  Issue:       {{ $json.issue }}\n  Status:      Open\n  Created At:  {{ $now.toFormat('dd/MM/yyyy HH:mm') }}\n  Resolved At: (empty)\n```\n\n### Slack Message Config\n```\nChannel: #it-support\nMessage:\n🎫 New IT Ticket\n\n*ID:* {{ $json.ticketId }}\n*From:* {{ $json.user }}\n*Issue:* {{ $json.issue }}\n*Time:* {{ $now.toFormat('dd MMM yyyy HH:mm') }}\n\nReply on Telegram to acknowledge.\n```\n\n### Optional AI Enhancement\n```\nADD between Edit Fields and the parallel branches:\n\nBasic LLM Chain:\n  Model: Ollama phi3:mini (local, free) OR Groq (free cloud)\n  Prompt: \"Classify this IT issue into one of: Hardware, Software,\n           Network, Access, Other.\n           Issue: {{ $json.issue }}\n           Reply with ONLY the category.\"\n  Output field: category\n\nThen route by category:\nSwitch node → Hardware/Software/Network/Access → different Slack channels\n\nFallback if AI fails:\nIf: category is empty → category = \"Unclassified\"\n```\n\n---\n\n## Template 2 — Daily Status Report\n\n### What it does\nEvery morning at 9 AM: fetch data from multiple sources, compile a report, send via Telegram and email.\n\n### Node Structure\n```\nSchedule Trigger (daily at 9:00 AM Asia/Kolkata)\n    ↓\n[PARALLEL DATA FETCH]\n├── HTTP Request: Get pending tickets (your API/Sheets)\n├── Google Sheets: Read open tasks\n└── HTTP Request: Get yesterday's metrics\n    ↓\nMerge (Append all data)\n    ↓\nCode Node (compile report)\n    ↓\nTelegram: Send daily report\n    ↓\nGmail: Send email report (with formatted HTML)\n```\n\n### Code Node (Report Compiler)\n```javascript\n// Get data from all merged branches\nconst allItems = $input.all();\n\n// Separate by source (add a source field in each HTTP Request node)\nconst tickets = allItems.filter(i => i.json.source === 'tickets');\nconst tasks = allItems.filter(i => i.json.source === 'tasks');\n\nconst date = new Date().toLocaleDateString('en-IN', {\n  timeZone: 'Asia/Kolkata',\n  weekday: 'long',\n  day: 'numeric',\n  month: 'long'\n});\n\nconst openTickets = tickets.filter(t => t.json.status === 'Open').length;\nconst pendingTasks = tasks.filter(t => !t.json.completed).length;\n\nconst telegramReport =\n  `📊 Daily Status — ${date}\\n\\n` +\n  `🎫 Open Tickets: ${openTickets}\\n` +\n  `✅ Pending Tasks: ${pendingTasks}\\n\\n` +\n  `Full details in Google Sheets.`;\n\nconst htmlReport =\n  `<h2>Daily Status — ${date}</h2>` +\n  `<p><b>Open Tickets:</b> ${openTickets}</p>` +\n  `<p><b>Pending Tasks:</b> ${pendingTasks}</p>`;\n\nreturn [{ json: { telegramReport, htmlReport, date } }];\n```\n\n---\n\n## Template 3 — Gmail to Google Sheets CRM\n\n### What it does\nWhen a new   ↓\nFilter: sender NOT in your contacts (avoid internal emails)\n    ↓\nEdit Fields (extract lead data)\n  name:    {{ $json.from.split('<')[0].trim() }}\n  email:   {{ submitted (leave request, expense, etc.), manager gets a Telegram message with Approve/Reject buttons. Decision is logged and applicant is notified.\n\n### Node Structure\n```\nn8n Form Trigger\n  Fields: Employee Name, Request Type, Details, Date\n    ↓\nEdit Fields\n  requestId: REQ-{{ $now.toMillis() }}\n  submittedAt: {{ $now.toISO() }}\n    ↓\nGoogle Sheets: Append Row (log request)\n    ↓\nTelegram: Send and Wait for Response\n  Chat ID: MANAGER_CHAT_ID\n  Message:\n  <b>📋 New Request</b>\n\n  <b>ID:</b> {{ $json.requestId }}\n  <b>From:</b> {{ $json.employeeName }}\n  <b>Type:</b> {{ $json.requestType }}\n  <b>Details:</b> {{ $json.details }}\n  <b>Date:</b> {{ $json.date }}\n  Response Type: Approval\n  Approval Options:\n    ✅ Approve\n    ❌ Reject\n  Limit Wait Time: 3 Days\n    ↓\nIf: response == 'approved'\n    ├── YES:\n    │   Google Sheets: Update Row (status = Approved)\n    │   Gmail: Send to employee \"Your request was APPROVED\"\n    │   Telegram (to employee): \"✅ Your {{ $json.requestType }} was approved!\"\n    └── NO:\n        Google Sheets: Update Row (status = Rejected)\n        Gmail: Send to employee \"Your request was REJECTED\"\n```\n\n---\n\n## Template 5 — Scheduled Database Sync\n\n### What it does\nEvery hour: fetch new/updated records from an external API, check if already in the database, insert or update as appropriate.\n\n### Node Structure\n```\nSchedule Trigger (every 1 hour)\n    ↓\nHTTP Request: GET API records updated in last 2 hours\n  URL: https://api.example.com/records?updated_after={{ $now.minus({hours:2}).toISO() }}\n    ↓\nSplit Out: split array into individual records\n    ↓\nLoop Over Items (batch size: 10)\n    ↓\n  MySQL/PostgreSQL: Check if record exists\n    Query: SELECT id FROM records WHERE external_id = :externalId\n    Parameters: externalId = {{ $json.id }}\n    Settings: On Error: Continue\n    ↓\n  If: record exists? (check if query returned rows)\n    ├── YES: UPDATE existing record\n    │   MySQL: UPDATE records SET name=:name, status=:status,\n    │          updated_at=NOW() WHERE external_id=:id\n    └── NO: INSERT new record\n        MySQL: INSERT INTO records (external_id, name, status, created_at)\n               VALUES (:id, :name, :status, NOW())\n    ↓\n  Wait: 1 second (rate limit protection)\n    ↓\n[Loop done output] → Telegram: \"Sync complete: X records processed\"\n```\n\n---\n\n## Template 6 — CareerForge Lead Qualifier (Pure Automation)\n\n### What it does\nNew form submission from CareerForge website → score the lead → route to right follow-up.\n\n### Node Structure\n```\nWebhook Trigger (from CareerForge form)\n  OR n8n Form Trigger\n    ↓\nEdit Fields (normalise data)\n    ↓\nCode Node (score the lead)\n    ↓\nSwitch (by score tier)\n  ├── Hot (score >= 80)     → HubSpot + immediate Telegram alert + personalised email\n  ├── Warm (score 50–79)   → HubSpot + add to email sequence\n  └── Cold (score < 50)    → HubSpot only (low priority)\n```\n\n### Code Node (Lead Scorer — No AI Needed)\n```javascript\nconst lead = $json;\nlet score = 0;\n\n// Score by job title seniority\nconst title = (lead.jobTitle || '').toLowerCase();\nif (title.includes('director') || title.includes('head')) score += 30;\nelse if (title.includes('manager') || title.includes('lead')) score += 20;\nelse if (title.includes('senior') || title.includes('architect')) score += 15;\nelse score += 5;\n\n// Score by experience\nconst years = parseInt(lead.yearsExperience) || 0;\nif (years >= 10) score += 25;\nelse if (years >= 5) score += 15;\nelse if (years >= 2) score += 5;\n\n// Score by current pain point\nconst pain = (lead.painPoint || '').toLowerCase();\nif (pain.includes('job') || pain.includes('salary') || pain.includes('career')) score += 20;\nelse if (pain.includes('skill') || pain.includes('upskill')) score += 10;\n\n// Score by region (UAE/GCC = target market)\nconst region = (lead.location || '').toLowerCase();\nif (region.includes('uae') || region.includes('dubai') || region.includes('gcc')) score += 25;\nelse if (region.includes('india')) score += 15;\n\nconst tier = score >= 80 ? 'Hot' : score >= 50 ? 'Warm' : 'Cold';\n\nreturn [{ json: { ...lead, score, tier } }];\n```\n\n### Optional AI Enhancement (Fallback Strategy)\n```\nAdd after Code Node:\n\nBasic LLM Chain (classify intent from message field):\n  Primary: Groq llama-3.3-70b-versatile (free, fast)\n  Prompt: \"Based on this message, what is this person's primary goal?\n           Message: {{ $json.message }}\n           Choose ONE: job-search | salary-negotiation | upskilling | career-change | other\"\n\n  If Groq 429 error → fallback to:\n  Ollama phi3:mini (local) OR Gemini free tier\n\n  On Error: Continue → use 'other' as default\n```\n\n---\n\n## Template 7 — Scheduled n8n Workflow Backup to GitHub\n\nSee full guide: [`workflows/01-save-workflows-to-github.md`](./01-save-workflows-to-github.md)\n\nQuick summary:\n```\nSchedule Trigger (daily 2 AM)\n    ↓\nHTTP Request: n8n API → get all workflows\n    ↓\nSplit Out → Loop Over Items\n    ↓\n  Code: prepare file name + base64 encode\n  GitHub: Get File (check if exists)\n  If exists: GitHub Edit | If new: GitHub Create\n    ↓\nTelegram: \"Backup complete: X workflows saved\"\n```\n
