# Save n8n Workflows to GitHub — Complete Guide\n\n> **Author:** Shrinivas Ramaprasad | **Updated:** May 2026\n> **Official template refs:**\n> - https://n8n.io/workflows/817-save-your-workflows-into-a-github-repository/\n> - https://n8n.io/workflows/1534-back-up-your-n8n-workflows-to-github/\n> - https://n8n.io/workflows/2532-backup-workflows-to-git-repository-on-github/\n> - https://n8n.io/workflows/4064-automated-daily-workflow-backup-to-github/\n\n---\n\n## YES — n8n Can Save Workflows Directly to GitHub\n\nThere are **two methods**:\n\n| Method | Who It's For | Requires |\n|---|---|---|\n| **Method 1: Source Control (built-in)** | n8n Enterprise plan | Enterprise license, Git config in Settings |\n| **Method 2: Backup Workflow (free)** | Everyone on free/community | n8n API key, GitHub PAT, GitHub node |\n\n**Recommendation:** Use Method 2 — it works on the free self-hosted version, is fully customisable, and is what most users actually use.\n\n---\n\n## Method 1 — Built-in Source Control (Enterprise Only)\n\n> **WHY:** Enterprise feature that gives every workflow built-in version control, with push/pull from Git without writing any automation.\n\n```\nAvailability: n8n Enterprise plan only\nDocs: https://docs.n8n.io/source-control-environments/\n\nSetup steps:\n1. n8n Settings → Source Control\n2. Connect your Git repository (GitHub, GitLab, etc.)\n3. Set up branches (e.g., main = production, staging = dev)\n4. Every workflow save can be pushed to Git\n5. Pull workflows from Git to another n8n instance\n\nLimitation: NOT available on free/community self-hosted n8n.\n```\n\n---\n\n## Method 2 — Workflow-Based Backup (Free, Works on All Plans)\n\n### HOW IT WORKS\n\n```\nSchedule Trigger (e.g., daily 2 AM)\n          ↓\nn8n API: GET /api/v1/workflows  ← Fetch all workflow JSON\n          ↓\nSplit Out node  ← One item per workflow\n          ↓\nLoop Over Items\n          ↓\n  GitHub node: Get File  ← Check if file already exists (get SHA)\n  (On Error: Continue)\n          ↓\n  If node: File exists?\n    ├── YES → GitHub: Edit File (update with current SHA)\n    └── NO  → GitHub: Create File\n          ↓\n[Done output] → Telegram/Slack: "Backed up X workflows"\n```\n\n---\n\n## Step-by-Step Setup\n\n### Step 1 — Enable the n8n API\n\n```\n1. In n8n: Settings (gear icon) → n8n API\n2. Toggle: Enable API\n3. Click: + Create an API key\n   Name: backup-key\n4. Copy the key — starts with: n8n_api_...\n5. Store in password manager\n\nWHY: The n8n API lets workflows call n8n itself to fetch all\nworkflow definitions as JSON.\n```\n\n### Step 2 — Create a GitHub Repository for Backups\n\n```\n1. github.com → + New repository\n2. Repository name: n8n-workflow-backups\n3. Visibility: Private (recommended — workflows may contain logic)\n4. Initialize with README: ✅\n5. Create repository\n\nFolder structure will be:\nn8n-workflow-backups/\n├── README.md\n└── workflows/\n    ├── Daily_Report_Generator.json\n    ├── IT_Support_Telegram_Bot.json\n    ├── CareerForge_Lead_Qualifier.json\n    └── ...\n```\n\n### Step 3 — Create GitHub PAT with Write Access\n\n```\n1. github.com → Settings → Developer settings\n   → Personal access tokens → Fine-grained tokens\n2. Generate new token:\n   Name: n8n-backup-bot\n   Expiration: 1 year\n   Repository access: Only select → n8n-workflow-backups\n   Permissions:\n     Contents: Read and Write  ← REQUIRED for creating/editing files\n     Metadata: Read-only\n3. Generate → Copy token (github_pat_...)\n4. In n8n: Settings → Credentials → + Add → GitHub\n   Name: GitHub - n8n Backups\n   Access Token: paste token → Save\n```\n\n### Step 4 — Build the Backup Workflow\n\nCreate a new workflow in n8n named **"n8n Workflow Auto-Backup to GitHub"**.\n\n#### Node 1 — Schedule Trigger\n```\nNode type: Schedule Trigger\nTrigger Interval: Every Day\nHour: 2\nMinute: 0\nTimezone: Asia/Kolkata\n\nWHY 2 AM: Low traffic time, no active workflow executions\n```\n\n#### Node 2 — HTTP Request (Fetch All Workflows)\n```\nNode type: HTTP Request\nMethod: GET\nURL: http://localhost:5678/api/v1/workflows\n  (or https://your-n8n-domain.com/api/v1/workflows if on VPS)\nAuthentication: Header Auth\n  Header Name: X-N8N-API-KEY\n  Header Value: n8n_api_your-key-here\nOptions:\n  Response Format: JSON\n\nOutput: { data: [ {workflow1}, {workflow2}, ... ] }\n```\n\n#### Node 3 — Edit Fields (Extract Workflow Array)\n```\nNode type: Edit Fields (Set)\nMode: Manual\nField:\n  Name: workflows\n  Value: {{ $json.data }}\n  Type: Array\n```\n\n#### Node 4 — Split Out (One Item Per Workflow)\n```\nNode type: Split Out\nField to Split Out: workflows\n\nNow each item = one workflow object\nEach item has: { id, name, active, nodes, connections, ... }\n```\n\n#### Node 5 — Code Node (Prepare File Content)\n```javascript\n// Prepare file path and base64 content for GitHub\nconst workflow = $json;\n\n// Clean filename: replace special chars with underscore\nconst safeName = workflow.name\n  .replace(/[^a-zA-Z0-9-_ ]/g, '_')\n  .replace(/\\s+/g, '_');\n\nconst fileName = `${safeName}_${workflow.id}.json`;\nconst filePath = `workflows/${fileName}`;\n\n// Format JSON with indentation for readable Git diffs\nconst content = JSON.stringify(workflow, null, 2);\n\n// GitHub API requires base64 encoded content\nconst base64Content = Buffer.from(content).toString('base64');\n\nreturn [{\n  json: {\n    filePath,\n    fileName,\n    base64Content,\n    workflowId: workflow.id,\n    workflowName: workflow.name,\n    isActive: workflow.active,\n    commitMessage: `backup: ${workflow.name} (${new Date().toISOString().split('T')[0]})`\n  }\n}];\n```\n\n#### Node 6 — GitHub: Get File (Check If Exists)\n```\nNode type: GitHub\nResource: File\nOperation: Get\nOwner: Shri-Phnx  (your GitHub username)\nRepository: n8n-workflow-backups\nFile Path: {{ $json.filePath }}\nBranch: main\n\nSettings Tab:\n  On Error: Continue   ← IMPORTANT: file won't exist on first run\n  Always Output Data: ON\n```\n\n#### Node 7 — Code Node (Extract SHA or Set Flag)\n```javascript\n// If file exists, GitHub returned a SHA we need for updating\n// If file doesn't exist, $json will have an error field\n\nconst hasFile = $json.sha !== undefined;\n\nreturn [{\n  json: {\n    ...$input.first().json,  // keep our prepared data\n    existingFileSha: hasFile ? $json.sha : null,\n    fileExists: hasFile\n  }\n}];\n```\n\n#### Node 8 — If (File Exists?)\n```\nNode type: If\nCondition: {{ $json.fileExists }} equals true\n\n[True output 0]  → Update existing file\n[False output 1] → Create new file\n```\n\n#### Node 9a — GitHub: Edit File (True branch)\n```\nNode type: GitHub\nResource: File\nOperation: Edit\nOwner: Shri-Phnx\nRepository: n8n-workflow-backups\nFile Path: {{ $json.filePath }}\nFile Content: {{ $json.base64Content }}\nCommit Message: {{ $json.commitMessage }}\nBranch: main\nFile SHA: {{ $json.existingFileSha }}  ← REQUIRED for edit\n\nWHY SHA: GitHub uses this to prevent conflicting overwrites.\nWithout it, the edit fails with a 409 Conflict error.\n```\n\n#### Node 9b — GitHub: Create File (False branch)\n```\nNode type: GitHub\nResource: File\nOperation: Create\nOwner: Shri-Phnx\nRepository: n8n-workflow-backups\nFile Path: {{ $json.filePath }}\nFile Content: {{ $json.base64Content }}\nCommit Message: {{ $json.commitMessage }}\nBranch: main\n```\n\n#### Node 10 — Merge (Rejoin Both Branches)\n```\nNode type: Merge\nMode: Append\n\nBoth true and false branch results merge here\n```\n\n#### Node 11 — Aggregate (Count Results)\n```javascript\n// Code Node (Run Once for All Items)\nconst items = $input.all();\nconst total = items.length;\nconst succeeded = items.filter(i => !i.json.error).length;\nconst failed = total - succeeded;\nconst date = new Date().toLocaleDateString('en-IN', {timeZone:'Asia/Kolkata'});\n\nreturn [{\n  json: {\n    message: `✅ n8n Backup Complete — ${date}\\n📁 ${succeeded}/${total} workflows backed up to GitHub\\n${failed > 0 ? `❌ ${failed} failed` : '🎉 All workflows saved'}`,\n    total,\n    succeeded,\n    failed,\n    repoUrl: 'https://github.com/Shri-Phnx/n8n-workflow-backups'\n  }\n}];\n```\n\n#### Node 12 — Telegram: Send Completion Message\n```\nNode type: Telegram\nResource: Message\nOperation: Send Message\nCredential: Telegram - IT Support Bot\nChat ID: your-chat-id\nText: {{ $json.message }}\nParse Mode: HTML\n```\n\n---\n\n## Restore Workflows From GitHub\n\n```\nTo restore a workflow from the GitHub backup:\n\n1. Go to: github.com/Shri-Phnx/n8n-workflow-backups/workflows/\n2. Find the workflow file → click → Raw\n3. Copy the JSON content\n4. In n8n: top-right menu → Import from URL or Import from File\n5. Paste/upload the JSON → Import\n6. Review and activate\n\nOr restore all at once using the n8n CLI:\n  n8n import:workflow --input=./workflows/ --separate\n```\n\n---\n\n## Trigger the Backup Manually (On Demand)\n\nAdd this as a second trigger or modify the workflow:\n\n```\n1. Add a second trigger: Manual Trigger\n2. Connect it to the same HTTP Request node\n3. Now you can run backup on-demand AND on schedule\n```\n\nOr trigger via webhook:\n```\n1. Add Webhook trigger (path: trigger-backup)\n2. Send a POST to: https://your-n8n.com/webhook/trigger-backup\n   with: { "secret": "your-secret" }\n3. If: secret matches → run backup\n```\n\n---\n\n## Common Errors and Fixes\n\n| Error | Cause | Fix |\n|---|---|---|\n| `401 Unauthorized` on API call | Wrong API key | Re-check n8n API key in credential |\n| `409 Conflict` on GitHub Edit | Missing or wrong SHA | Use Get File before Edit to get current SHA |\n| `404` on GitHub Get File | File doesn't exist (first run) | Set On Error: Continue on Get File node |\n| `422 Unprocessable Entity` | Content not base64 encoded | Use `Buffer.from(text).toString('base64')` |\n| `403 Forbidden` on GitHub | PAT missing Contents: Write permission | Edit PAT on GitHub → add write permission |\n| Empty workflows returned | API not enabled | Settings → n8n API → Enable |\n| Commit shows garbled content | Not formatted as JSON | Use `JSON.stringify(workflow, null, 2)` |\n\n---\n\n## What Gets Saved\n\nEach `.json` file contains the complete workflow definition:\n```json\n{\n  "id": "wf_abc123",\n  "name": "Daily Report Generator",\n  "active": true,\n  "nodes": [ ... all nodes with config ... ],\n  "connections": { ... all node connections ... },\n  "settings": { "timezone": "Asia/Kolkata" },\n  "tags": []\n}\n```\n\nThis is everything needed to fully restore the workflow, **except** credentials (which are stored encrypted in n8n's database, not in workflow JSON — by design, for security).\n\n---\n\n## Git History = Workflow Version Control\n\nEvery time the backup runs, GitHub creates a new commit for each changed workflow. This gives you:\n\n- Full history of every change to every workflow\n- Ability to see exactly what changed (GitHub diff view)\n- Ability to restore any previous version by copying older JSON\n- A searchable archive of all your automation logic\n\nTo see what changed in a workflow:\n```\ngithub.com → n8n-workflow-backups → workflows/\n→ Click a workflow file → History\n→ Click any commit → See exact JSON diff\n```\n
