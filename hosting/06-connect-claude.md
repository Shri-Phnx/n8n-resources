# Connect n8n with Claude (Anthropic)\n\n> **Author:** Shrinivas Ramaprasad | **Updated:** May 2026\n> How to use Claude as an LLM in n8n, and how to expose n8n as an MCP server for Claude Desktop.\n\n---\n\n## Overview: Three Ways to Connect n8n and Claude\n\n| Method | What it does | Best for |\n|---|---|---|\n| **Method 1: Claude as LLM in n8n** | Use Claude (Anthropic API) as the AI brain inside n8n AI Agent or LLM Chain | Building AI automation workflows with Claude's intelligence |\n| **Method 2: n8n as MCP server for Claude** | Expose n8n workflows as tools that Claude Desktop can call | Letting Claude trigger your automations from chat |\n| **Method 3: Claude Desktop → n8n webhook** | Claude sends HTTP requests to n8n webhooks | Simple integration without MCP setup |\n\n---\n\n## Method 1 — Use Claude as LLM Inside n8n Workflows\n\n### WHY: Claude is one of the best LLMs for structured output, reasoning, and following complex instructions — ideal for n8n automation.\n\n### Step 1 — Get Anthropic API Key\n\n```\n1. Go to: https://console.anthropic.com\n2. Sign in / create account\n3. Left sidebar → API Keys → + Create Key\n   Name: n8n-automation\n4. Copy the key — starts with: sk-ant-api03-...\n   ⚠️ Shown only once — save to password manager\n```\n\n### Step 2 — Add Anthropic Credential in n8n\n\n```\n1. n8n → Settings (gear icon) → Credentials → + Add Credential\n2. Search: Anthropic\n3. Credential Name: Anthropic - Claude Prod\n4. API Key: sk-ant-api03-your-key\n5. Save → ✅ Connected\n```\n\n### Step 3 — Add Claude to a Workflow\n\n**Option A — AI Agent (Claude as reasoning engine with tools):**\n\n```\n1. Add AI Agent node to your workflow\n2. Click the Chat Model port (bottom of AI Agent)\n3. Search: Anthropic Chat Model → add it\n4. Configure Anthropic Chat Model:\n   Credential: Anthropic - Claude Prod\n   Model: claude-sonnet-4-6  ← best balance of quality/speed/cost\n         claude-opus-4-6    ← most capable, higher cost\n         claude-haiku-4-5   ← fastest, lowest cost\n   Max Tokens: 1024 (for short tasks) to 4096 (for complex outputs)\n   Temperature: 0 (deterministic) to 1 (creative). 0.3–0.5 for automation.\n5. Add system prompt to AI Agent:\n   \"You are an IT automation assistant. Today is {{ $now.toFormat('dd MMMM yyyy') }}.\n   You help users create support tickets and look up asset information.\n   Always confirm before making any changes.\"\n6. Optionally add:\n   - Memory sub-node (Redis Chat Memory for persistent conversations)\n   - Tool sub-nodes (HTTP Request Tool, Calculator, etc.)\n```\n\n**Option B — Basic LLM Chain (Claude for simple text tasks):**\n\n```\n1. Add Basic LLM Chain node\n2. Add Anthropic Chat Model sub-node\n3. Prompt: \"Classify this support ticket into one of: Hardware, Software, Network, Access\n           Ticket: {{ $json.ticketText }}\n           Reply with ONLY the category name.\"\n4. Output: clean category string for routing\n```\n\n### Available Claude Models in n8n\n\n| Model ID | Speed | Cost | Context | Best For |\n|---|---|---|---|---|\n| `claude-haiku-4-5-20251001` | Fastest | Cheapest | 200K | Simple classification, quick tasks |\n| `claude-sonnet-4-6` | Balanced | Medium | 200K | Most automation tasks |\n| `claude-opus-4-6` | Slowest | Most | 200K | Complex reasoning, analysis |\n\n---\n\n## Method 2 — n8n as MCP Server (Claude Desktop Calls n8n)\n\n### WHY: This lets you type in Claude Desktop \"Create an IT ticket for my VPN issue\" and Claude automatically calls your n8n workflow to do it.\n\n### How MCP Works with n8n\n\n```\nYou type in Claude Desktop:\n\"Search ServiceNow for my open incidents\"\n\nClaude recognises this matches the n8n MCP tool \"search_servicenow\"\n          ↓\nClaude calls: POST https://your-n8n.com/webhook/mcp-search\n{ \"query\": \"open incidents\", \"user\": \"shrinivas\" }\n          ↓\nn8n workflow runs (MCP Server Trigger)\n          ↓\nn8n calls ServiceNow API\n          ↓\nn8n returns results to Claude\n          ↓\nClaude shows you the results in chat\n```\n\n### Step 1 — Create an n8n MCP Workflow\n\n```\n1. Create new workflow: \"MCP: Search ServiceNow\"\n2. Add MCP Server Trigger node (find it under \"Other ways\")\n3. Configure MCP Server Trigger:\n   Tool Name: search_servicenow\n   Tool Description: Search ServiceNow for incidents, assets, or requests.\n                     Input: { query: string, type: \"incident\"|\"asset\"|\"request\" }\n                     Returns: list of matching records\n4. Add your workflow logic:\n   HTTP Request → ServiceNow API\n5. The last node's output is what Claude receives\n6. Activate the workflow\n```\n\n### Step 2 — Get the MCP Endpoint URL\n\n```\nWith the workflow active, the MCP endpoint is:\nhttps://your-n8n.com/mcp/workflow/{WORKFLOW_ID}/sse\n\nTo find WORKFLOW_ID:\n→ Open the workflow → look at the URL:\n  https://your-n8n.com/workflow/wf_abc123\n                                ↑\n                          this is the ID\n```\n\n### Step 3 — Configure Claude Desktop\n\n```\n1. Find Claude Desktop config file:\n   macOS: ~/Library/Application Support/Claude/claude_desktop_config.json\n   Windows: %APPDATA%\\Claude\\claude_desktop_config.json\n\n2. Edit the file (create if it doesn't exist):\n```\n\n```json\n{\n  \"mcpServers\": {\n    \"n8n-automation\": {\n      \"command\": \"npx\",\n      \"args\": [\n        \"mcp-remote\",\n        \"https://your-n8n.com/mcp/workflow/wf_abc123/sse\"\n      ]\n    }\n  }\n}\n```\n\n```\n3. Save the file\n4. Restart Claude Desktop\n5. You'll see a hammer icon (🔨) in Claude Desktop\n   → Click it → your n8n tools appear\n6. Test: type \"search servicenow for open incidents\"\n   → Claude calls your n8n workflow → returns results\n```\n\n### Step 4 — Add Authentication to the MCP Endpoint\n\n```\nFor security, add a header auth to the MCP Server Trigger:\n\n1. In the MCP Server Trigger node → Authentication → Header Auth\n2. Create credential:\n   Header Name: X-MCP-Secret\n   Header Value: generate with: openssl rand -hex 32\n3. In Claude Desktop config, add the header:\n```\n\n```json\n{\n  \"mcpServers\": {\n    \"n8n-automation\": {\n      \"command\": \"npx\",\n      \"args\": [\n        \"mcp-remote\",\n        \"https://your-n8n.com/mcp/workflow/wf_abc123/sse\",\n        \"--header\",\n        \"X-MCP-Secret: your-secret-here\"\n      ]\n    }\n  }\n}\n```\n\n### Multiple n8n Tools for Claude Desktop\n\n```json\n{\n  \"mcpServers\": {\n    \"n8n-it-tools\": {\n      \"command\": \"npx\",\n      \"args\": [\"mcp-remote\", \"https://your-n8n.com/mcp/workflow/wf_TICKETING/sse\"]\n    },\n    \"n8n-careerforge\": {\n      \"command\": \"npx\",\n      \"args\": [\"mcp-remote\", \"https://your-n8n.com/mcp/workflow/wf_CAREERFORGE/sse\"]\n    },\n    \"n8n-reports\": {\n      \"command\": \"npx\",\n      \"args\": [\"mcp-remote\", \"https://your-n8n.com/mcp/workflow/wf_REPORTS/sse\"]\n    }\n  }\n}\n```\n\n---\n\n## Method 3 — Simple Webhook (No MCP Needed)\n\n### WHY: If you just want Claude to sometimes trigger a workflow, a simple webhook is faster to set up than full MCP.\n\n```\n1. In n8n: Add Webhook trigger\n   Path: claude-action\n   Method: POST\n   Auth: Header Auth (X-Claude-Secret: your-secret)\n   Activate workflow\n\n2. In Claude: Use the HTTP tool (if available) or tell Claude:\n   \"Make a POST request to https://your-n8n.com/webhook/claude-action\n    with header X-Claude-Secret: your-secret\n    and body: { 'action': 'create_ticket', 'description': 'VPN not working' }\"\n\n3. Claude Desktop with Fetch MCP can do this directly:\n   claude_desktop_config.json → add mcp-fetch server\n```\n\n---\n\n## Use Cases: Claude + n8n\n\n| Use Case | How |\n|---|---|---|\n| AI support ticket bot | Telegram Trigger → AI Agent (Claude) → ServiceNow | Claude classifies and routes |\n| CV tailoring | Webhook → Claude (Basic LLM Chain) → Google Docs | Claude rewrites CV |\n| Email triage | Gmail Trigger → Claude (classify) → route to folder/CRM | Claude reads and categorises |\n| Meeting notes | Form Trigger (paste notes) → Claude (summarise + action items) → Notion | Claude structures output |\n| CareerForge lead qualifier | Form Trigger → Claude (score and categorise) → HubSpot | Claude assesses fit |\n| Recruiter outreach | Spreadsheet data → Claude (personalise) → LinkedIn draft | Claude personalises messages |\n\n---\n\n## Cost Optimisation Tips\n\n```\n1. Use Haiku for simple classification (10x cheaper than Sonnet)\n2. Use Sonnet for most automation tasks (best value)\n3. Use Opus only for complex analysis or high-stakes decisions\n4. Set Temperature: 0 for automation (consistent outputs, no wasted tokens)\n5. Keep system prompts concise — every token costs\n6. Test with Groq free tier first, switch to Claude when logic is proven\n7. Monitor usage at console.anthropic.com → Usage tab\n```\n\n---\n\n## Troubleshooting\n\n| Problem | Cause | Fix |\n|---|---|---|\n| `401 Unauthorized` | API key wrong or expired | Check console.anthropic.com → regenerate key |\n| `529 Overloaded` | Anthropic API busy | Enable Retry On Fail: 3 tries, 10s wait |\n| `400 Bad Request` | Max tokens too high | Check model's max output limit |\n| MCP tools not showing in Claude Desktop | Config file wrong | Validate JSON syntax, restart Claude Desktop |\n| MCP endpoint not reachable | n8n not accessible from internet | Need VPS or ngrok — localhost won't work for Claude Desktop MCP |\n| Tool description confuses Claude | Vague description | Be very specific: input format, output format, when to use |\n
