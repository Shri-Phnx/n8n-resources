Purchase a Hostinger KVM 2 plan\n2. After purchase → VPS dashboard appears\n3. Set root password OR add SSH key (SSH key recommended)\n4. Note your VPS IP address: e.g., 85.123.45.67\n5. Connect via SSH:\n\n   Windows:\n     → Download PuTTY: https://www.putty.org\n     → Host: 85.123.45.67 | Port: 22 | Click Open\n     → Login as: root | Enter password\n\n   macOS / Linux:\n     → Open Terminal\n     → ssh root@85.123.45.67\n     → Type password when prompted\n\n   You're now inside your VPS\n```\n\n### Step 2 — Initial Server Security\n\n```bash\n# Update all packages\napt update && apt upgrade -y\n\n# Create a non-root user (running as root long-term is risky)\nadduser n8n-user\n# Enter a password when prompted\nusermod -aG sudo n8n-user\n\n# Set up firewall\nufw allow 22     # SSH — ALWAYS do this first or you'll lock yourself out\nufw allow 80     # HTTP (needed for SSL cert verification)\nufw allow 443    # HTTPS (your n8n URL)\nufw enable\n# Type 'y' when asked\n\nufw status\n# Should show: Status active, ports 22, 80, 443 allowed\n\n# Reconnect as new user\nexit\nssh n8n-user@85.123.45.67\n```\n\n### Step 3 — Install Docker\n\n```bash\n# Install Docker\ncurl -fsSL https://get.docker.com | sh\n\n# Add user to docker group (no sudo needed for docker commands)\nsudo usermod -aG docker $USER\nnewgrp docker\n\n# Install Docker Compose plugin\nsudo apt-get install -y docker-compose-plugin\n\n# Verify\ndocker --version\ndocker compose version\ndocker run hello-world\n```\n\n### Step 4 — Point a Domain to Your VPS\n\n**WHY:** You need a domain for HTTPS. HTTPS is required for OAuth (Google), for Telegram webhook mode, and for general security.\n\n```\nOption A — Buy domain from Hostinger (easiest):\n  Hostinger dashboard → Domains → Buy domain\n  e.g., yourdomain.com or n8n.yourdomain.com\n\nOption B — Use your existing domain (any registrar):\n  Go to your domain's DNS settings\n  Add an A Record:\n    Name: n8n  (or @ for root domain)\n    Type: A\n    Value: 85.123.45.67  (your VPS IP)\n    TTL: 3600\n\nWait 5–30 minutes for DNS to propagate\n\nVerify:\n  ping n8n.yourdomain.com\n  → Should reply from 85.123.45.67\n\nOption C — Free subdomain with ngrok (no domain purchase needed):\n  See hosting/03-docker-ngrok-guide.md\n```\n\n### Step 5 — Create n8n Project\n\n```bash\nmkdir ~/n8n && cd ~/n8n\n\n# Generate encryption key (run this ONCE, save the output in password manager)\nopenssl rand -hex 32\n# Example: a8f3b2c1d4e5f6a7b8c9d0e1f2a3b4c5d6e7f8a9b0c1d2e3f4a5b6c7d8e9f0a1\n\n# Create .env file\nnano .env\n```\n\nPaste this into the `.env` file (Ctrl+X → Y → Enter to save in nano):\n\n```env\n# n8n Configuration — Production VPS\n# Created: 2026-05-04\n\n# ===== SECURITY =====\n# Generate with: openssl rand -hex 32\n# NEVER change this after first run\nN8N_ENCRYPTION_KEY=paste-your-32-char-key-here\n\n# Login for n8n UI\nN8N_BASIC_AUTH_ACTIVE=true\nN8N_BASIC_AUTH_USER=admin\nN8N_BASIC_AUTH_PASSWORD=YourVeryStrongPassword!\n\n# ===== NETWORK =====\nN8N_HOST=0.0.0.0\nN8N_PORT=5678\nN8N_PROTOCOL=https\n# Replace with your actual domain:\nWEBHOOK_URL=https://n8n.yourdomain.com\nN8N_EDITOR_BASE_URL=https://n8n.yourdomain.com\n\n# ===== TIMEZONE =====\nGENERIC_TIMEZONE=Asia/Kolkata\nTZ=Asia/Kolkata\n\n# ===== DATABASE (switch to PostgreSQL for production) =====\nDB_TYPE=postgresdb\nDB_POSTGRESDB_HOST=postgres\nDB_POSTGRESDB_PORT=5432\nDB_POSTGRESDB_DATABASE=n8n\nDB_POSTGRESDB_USER=n8n\nDB_POSTGRESDB_PASSWORD=AnotherStrongDBPassword!\n\n# ===== EXECUTION =====\nEXECUTIONS_DATA_MAX_AGE=168\n```\n\n### Step 6 — Create docker-compose.yml\n\n```bash\nnano docker-compose.yml\n```\n\n```yaml\nversion: '3.8'\n\nservices:\n  # ── PostgreSQL database ──────────────────────────────────\n  postgres:\n    image: postgres:16\n    container_name: postgres\n    restart: unless-stopped\n    environment:\n      POSTGRES_DB: n8n\n      POSTGRES_USER: n8n\n      POSTGRES_PASSWORD: ${DB_POSTGRESDB_PASSWORD}\n    volumes:\n      - postgres_data:/var/lib/postgresql/data\n    healthcheck:\n      test: [\"CMD-SHELL\", \"pg_isready -U n8n\"]\n      interval: 5s\n      timeout: 5s\n      retries: 5\n\n  # ── n8n automation engine ────────────────────────────────\n  n8n:\n    image: docker.n8n.io/n8nio/n8n:latest\n    container_name: n8n\n    restart: unless-stopped\n    expose:\n      - \"5678\"\n      # expose (not ports) because nginx handles external traffic\n    env_file:\n      - .env\n    volumes:\n      - n8n_data:/home/node/.n8n\n    depends_on:\n      postgres:\n        condition: service_healthy\n\n  # ── nginx reverse proxy with SSL ────────────────────────\n  nginx:\n    image: nginx:alpine\n    container_name: nginx\n    restart: unless-stopped\n    ports:\n      - \"80:80\"\n      - \"443:443\"\n    volumes:\n      - ./nginx:/etc/nginx/conf.d\n      - ./ssl:/etc/letsencrypt\n      - ./certbot-www:/var/www/certbot\n    depends_on:\n      - n8n\n\n  # ── certbot for SSL certificate ─────────────────────────\n  certbot:\n    image: certbot/certbot\n    container_name: certbot\n    volumes:\n      - ./ssl:/etc/letsencrypt\n      - ./certbot-www:/var/www/certbot\n    entrypoint: \"/bin/sh -c 'trap exit TERM; while :; do certbot renew; sleep 12h & wait $${!}; done;'\"\n\nvolumes:\n  postgres_data:\n  n8n_data:\n```\n\n### Step 7 — Configure nginx\n\n```bash\nmkdir nginx\nnano nginx/n8n.conf\n```\n\n```nginx\n# HTTP — redirect to HTTPS and handle SSL cert verification\nserver {\n    listen 80;\n    server_name n8n.yourdomain.com;\n\n    # Let's Encrypt certificate verification\n    location /.well-known/acme-challenge/ {\n        root /var/www/certbot;\n    }\n\n    # Redirect all other traffic to HTTPS\n    location / {\n        return 301 https://$host$request_uri;\n    }\n}\n\n# HTTPS — main server block\nserver {\n    listen 443 ssl;\n    server_name n8n.yourdomain.com;\n\n    # SSL certificate files (created by certbot)\n    ssl_certificate /etc/letsencrypt/live/n8n.yourdomain.com/fullchain.pem;\n    ssl_certificate_key /etc/letsencrypt/live/n8n.yourdomain.com/privkey.pem;\n\n    # Security headers\n    add_header Strict-Transport-Security \"max-age=31536000; includeSubDomains\" always;\n\n    location / {\n        proxy_pass http://n8n:5678;\n        proxy_http_version 1.1;\n\n        # WebSocket support (required for n8n editor)\n        proxy_set_header Upgrade $http_upgrade;\n        proxy_set_header Connection 'upgrade';\n\n        # Pass real client IP to n8n\n        proxy_set_header Host $host;\n        proxy_set_header X-Real-IP $remote_addr;\n        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;\n        proxy_set_header X-Forwarded-Proto $scheme;\n\n        # Long timeout for AI operations and approval workflows\n        proxy_read_timeout 86400s;\n        proxy_send_timeout 86400s;\n\n        # Max file upload size\n        client_max_body_size 100M;\n    }\n}\n```\n\n### Step 8 — Get SSL Certificate\n\n```bash\n# Start nginx first (needed for certificate verification)\ndocker compose up -d nginx\n\n# Get SSL certificate\ndocker compose run --rm certbot certonly --webroot \\\n  -w /var/www/certbot \\\n  -d n8n.yourdomain.com \\\n  --email your@email.com \\\n  --agree-tos \\\n  --no-eff-email\n\n# Expected output:\n# Successfully received certificate.\n# Certificate saved at: /etc/letsencrypt/live/n8n.yourdomain.com/\n\n# Start everything\ndocker compose up -d\n\n# Check status\ndocker compose ps\n# All containers should show: Up\n```\n\n### Step 9 — Access n8n\n\nOpen browser: **https://n8n.yourdomain.com**\n\nFirst login: 22 | Use VPS provider's web console → `ufw allow 22` |\n| `502 Bad Gateway` on browser | n8n not started | `docker compose ps` → `docker compose logs n8n` |\n| `SSL certificate not found` | Certbot didn't run or domain DNS not propagated | Wait 30 min → re-run certbot command |\n| Webhook URL still shows localhost | `WEBHOOK_URL` not set in .env | Edit .env → `docker compose restart n8n` |\n| n8n credentials can't decrypt | `N8N_ENCRYPTION_KEY` changed | Restore original key in .env |\n| Disk full | Docker images + logs | `docker system prune -a` |\n| VPS reboots, n8n not starting | `restart: unless-stopped` missing | Add it to docker-compose.yml → redeploy |\n
