# Trigger Nodes + ngrok — How to Connect Messaging Apps to n8n\n\n> **Author:** Shrinivas Ramaprasad | **Updated:** May 2026  \n> **Official n8n Triggers docs:** https://docs.n8n.io/integrations/builtin/core-nodes/n8n-nodes-base.webhook/  \n> **Telegram Bot API:** https://core.telegram.org/bots/api  \n> **WhatsApp Business API:** https://developers.facebook.com/docs/whatsapp\n\n---\n\n## WHY This Guide Exists\n\nWhen you run n8n locally with Docker, your n8n is at `http://localhost:5678`. This is a private address — only your machine can reach it. Services like Telegram, Slack, Discord, and GitHub need to **push events to a public URL**. Without a public URL, these triggers simply won't fire.\n\n**The solution:** Use ngrok (or Cloudflare Tunnel) to get a public HTTPS URL, then point your trigger nodes at that URL.\n\n**Reference for Docker + ngrok setup:** `hosting/03-docker-ngrok-guide.md`\n\n---\n\n## Which Triggers Need a Public URL?\n\n| Trigger | Needs Public URL | Why | Works Locally? |\n|---|---|---|---|\n| **Telegram Trigger** | ✅ YES (webhook mode) | Telegram pushes messages to your URL | ✅ Polling mode works locally |\n| **Slack Trigger** | ✅ YES | Slack Events API pushes to your URL | ❌ No polling mode |\n| **Discord Trigger** | ✅ YES | Discord pushes events to your URL | ❌ No polling mode |\n| **WhatsApp Trigger** | ✅ YES | Meta API pushes to your webhook | ❌ No polling mode |\n| **GitHub Trigger** | ✅ YES | GitHub pushes events to your webhook | ❌ No polling mode |\n| **Stripe Trigger** | ✅ YES | Stripe sends payment events | ❌ No polling mode |\n| **Webhook Trigger** | ✅ YES | General inbound HTTP | ❌ Not externally |\n| **Gmail Trigger** | ❌ No | n8n polls Gmail periodically | ✅ Yes — polling |\n| **Schedule Trigger** | ❌ No | n8n runs on a timer | ✅ Yes |\n| **Manual Trigger** | ❌ No | You click the button | ✅ Yes |\n\n---\n\n## Architecture: How Triggers Work with ngrok\n\n```\n  ┌─────────────────────────────────────────────────────────┐\n  │              Your Machine (Laptop / PC / NAS)           │\n  │                                                         │\n  │  ┌──────────────┐     ┌──────────────────────────────┐ │\n  │  │ ngrok agent  │────▶│  Docker: n8n (port 5678)     │ │\n  │  │ (forwarding) │     │  Webhook URL registered with │ │\n  │  └──────┬───────┘     │  Telegram / Slack / GitHub   │ │\n  │         │             └──────────────────────────────┘ │\n  └─────────┼───────────────────────────────────────────────┘\n            │ (HTTPS tunnel)\n  ┌─────────▼───────────────────────────────────────────────┐\n  │  ngrok Cloud: https://abc123.ngrok-free.app             │\n  │  (public HTTPS URL that anyone can reach)               │\n  └─────────▲───────────────────────────────────────────────┘\n            │\n  ┌─────────┼─────────────────────────────────────────────┐\n  │  External Services                                    │\n  │  Telegram: POST https://abc123.ngrok-free.app/webhook │\n  │  Slack:    POST https://abc123.ngrok-free.app/webhook │\n  │  Stripe:   POST https://abc123.ngrok-free.app/webhook │\n  └───────────────────────────────────────────────────────┘\n```\n\n---\n\n## Part 1 — Telegram Trigger: Complete Setup\n\n### WHY Telegram for n8n?\n- Free, no rate limits for personal bots\n- Native n8n node (no HTTP Request needed)\n- Supports `Send and Wait for Response` (approvals in Telegram)\n- Works as chatbot, notification sender, command handler\n- **Recommendation: #1 choice for personal and small-team automation**\n\n### Two modes: Polling vs Webhook\n\n| Mode | How it works | Needs ngrok? | Latency |\n|---|---|---|---|\n| **Polling** (default) | n8n checks Telegram every few seconds | ❌ No | ~1-3 seconds delay |\n| **Webhook** | Telegram pushes immediately to your URL | ✅ Yes | Instant |\n\nFor testing: use polling (no setup needed).  \nFor production: use webhook (instant, more efficient).\n\n### Step 1 — Create Telegram Bot\n\n```\n1. Open Telegram → search @BotFather\n2. Send: /newbot\n3. Enter display name: My n8n Bot\n4. Enter username: my_n8n_automation_bot\n   Rules:\n   ✅ Must end in 'bot'\n   ✅ Must be globally unique\n   ✅ Only letters, numbers, underscores\n   ❌ No spaces\n5. BotFather replies with your token:\n   1234567890:AAFxyz-AbCdEfGhIjKlMnOpQrStUvWxYz\n   Format: {10-digit-number}:{35-char-random-string}\n   ⚠️ Save this token immediately\n6. Optional customisation:\n   /setdescription → set bot description\n   /setuserpic     → set bot profile photo\n   /setcommands    → define /commands\n```\n\n### Step 2 — Configure Telegram Credential in n8n\n\n```\n1. n8n: Settings (gear) → Credentials → + Add Credential\n2. Search: Telegram\n3. Credential Name: Telegram - IT Support Bot\n4. Access Token: paste your token\n5. Click Save\n6. Test button should show ✅ Connected\n   If ❌: token has spaces or was copied wrong\n```\n\n### Step 3a — Telegram Trigger in POLLING mode (no ngrok needed)\n\n```\nIn n8n:\n1. New workflow → + Add node → search \"Telegram Trigger\"\n2. Configure:\n   Credential: Telegram - IT Support Bot\n   Updates: Message ← what to listen for\n   Additional Fields:\n     - Download Images: ON (to handle photo messages)\n     - Download Files: ON (to handle document messages)\n3. Click \"Execute workflow\" to start polling\n4. Send a message to your bot in Telegram\n5. n8n shows the captured message within ~3 seconds\n```\n\n### Step 3b — Telegram Trigger in WEBHOOK mode (instant, needs ngrok)\n\n```\nPrerequisite: ngrok running, WEBHOOK_URL updated in .env\nSee: hosting/03-docker-ngrok-guide.md → Part 3\n\nIn n8n:\n1. Add Telegram Trigger node\n2. Additional Fields → Webhook:\n   ✅ Enable webhook mode\n   The node will auto-register with Telegram using your WEBHOOK_URL\n3. Activate the workflow (toggle Active in editor)\n4. Send a message to your bot — arrives INSTANTLY\n```\n\n### Step 4 — Get your Chat ID\n\n```\nMethod 1 — From workflow output:\nWhen trigger fires, output contains:\n  {{ $json.message.chat.id }}\n  Copy this number — it's your Chat ID\n\nMethod 2 — From Telegram API:\n1. Send /start to your bot\n2. Open browser: https://api.telegram.org/bot{TOKEN}/getUpdates\n3. Look for: result[0].message.chat.id\n\nChat ID types:\n  123456789    = Personal/private chat (positive number)\n  -987654321   = Group chat (negative number)\n  -1001234567  = Channel (starts with -100)\n```\n\n### Telegram Trigger Output Fields\n\n```json\n{\n  \"update_id\": 123456789,\n  \"message\": {\n    \"message_id\": 42,\n    \"from\": {\n      \"id\": 987654321,\n      \"first_name\": \"Shrinivas\",\n      \"last_name\": \"Ramaprasad\",\n      \"username\": \"shrinivas_r\"\n    },\n    \"chat\": {\n      \"id\": 987654321,\n      \"type\": \"private\"\n    },\n    \"date\": 1714456800,\n    \"text\": \"/ticket My laptop won't start\"\n  }\n}\n```\n\nAccess: `{{ $json.message.chat.id }}` `{{ $json.message.text }}` `{{ $json.message.from.first_name }}`\n\n### Complete Telegram Bot Workflow Example\n\n```\nTelegram Trigger (listen for messages)\n    │\n    ├── If: text starts with /ticket\n    │       │\n    │       └── Edit Fields: build ticket\n    │               ticketId: TICK-{{ $now.toMillis() }}\n    │               user: {{ $json.message.from.first_name }}\n    │               issue: {{ $json.message.text.replace('/ticket ','') }}\n    │               chatId: {{ $json.message.chat.id }}\n    │           ↓\n    │           Google Sheets: Append row\n    │           ↓\n    │           Slack: Notify #it-support\n    │           ↓\n    │           Telegram: Send Message\n    │             Chat ID: {{ $json.chatId }}\n    │             Text: ✅ Ticket #{{ $json.ticketId }} created!\n    │\n    └── Else: Telegram: Send Message\n              Text: Unknown command. Use /ticket <describe issue>\n```\n\n**⚠️ Telegram mistakes:**\n| Mistake | Effect | Fix |\n|---|---|---|\n| User never started bot | Cannot send message | User must send /start first |\n| Wrong Chat ID sign | Message to wrong person | Check: groups=negative, personal=positive |\n| Bot not admin in channel | Can't post | Channel Settings → Admins → add bot |\n| Bot not added to group | Can't receive/send | Add bot: group settings → Add members → search bot |\n| HTML tags unclosed | Message error | Always close: `<b>text</b>` not `<b>text` |\n\n---\n\n## Part 2 — Slack Trigger: Complete Setup\n\n### WHY Slack?\n- Best for corporate/team environments already using Slack\n- Rich message blocks with buttons\n- Good for DevOps and engineering team notifications\n\n### Step 1 — Create Slack App\n\n```\n1. Go to: https://api.slack.com/apps\n2. Click: Create New App → From scratch\n3. App Name: n8n Automation\n4. Workspace: select your Slack workspace\n5. Click: Create App\n```\n\n### Step 2 — Enable Event Subscriptions (needs public URL)\n\n```\nPrerequisite: ngrok running with WEBHOOK_URL set in n8n\n\n1. Slack App settings → Event Subscriptions → toggle ON\n2. Request URL field:\n   Paste: https://abc123.ngrok-free.app/webhook/slack-events\n   (this is a webhook trigger URL you create in n8n)\n\nIMPORTANT: Slack will send a challenge request to verify your URL\nYou must respond to it. In n8n:\n   → Add Webhook trigger node\n   → Path: slack-events\n   → Respond: Using Respond to Webhook Node\n   → Add If node: check if $json.body.type == 'url_verification'\n       [true]  → Respond to Webhook: { \"challenge\": \"{{ $json.body.challenge }}\" }\n       [false] → continue to event processing\n\n3. Subscribe to Bot Events:\n   - message.channels  (messages in channels)\n   - app_mention       (when someone @mentions bot)\n   - reaction_added    (emoji reactions)\n4. Save Changes\n5. Reinstall app to workspace\n```\n\n### Step 3 — Configure in n8n\n\n```\n1. Settings → Credentials → + Add → Slack\n   Access Token: xoxb-your-token → Save\n2. Add Slack Trigger node (or use Webhook for events)\n3. Events: select message.channels, app_mention\n```\n\n---\n\n## Part 3 — Discord Trigger: Complete Setup\n\n### WHY Discord?\n- Free for community and developer projects\n- Good for gaming communities and open source projects\n- Simpler to set up than Slack for small teams\n\n### Method A: Discord Webhook (Outbound — Send to Discord)\n\n```\nSimplest method — no bot token needed\n\n1. Discord server → channel → Edit Channel → Integrations → Webhooks\n2. New Webhook → give it a name → Copy Webhook URL\n3. In n8n: HTTP Request node\n   Method: POST\n   URL: https://discord.com/api/webhooks/YOUR_WEBHOOK_URL\n   Body (JSON):\n   {\n     \"content\": \"{{ $json.message }}\",\n     \"username\": \"n8n Bot\",\n     \"embeds\": [{\n       \"title\": \"{{ $json.title }}\",\n       \"description\": \"{{ $json.description }}\",\n       \"color\": 3066993\n     }]\n   }\n```\n\n### Method B: Discord Bot (Receive messages from Discord)\n\n```\nStep 1 — Create Discord Application:\n1. Go to: https://discord.com/developers/applications\n2. New Application → Name: n8n Bot\n3. Left menu → Bot → Add Bot\n4. Token section: Reset Token → copy\n5. Privileged Gateway Intents:\n   ✅ Message Content Intent\n   ✅ Server Members Intent\n6. OAuth2 → URL Generator:\n   Scopes: bot\n   Bot Permissions: Send Messages, Read Messages\n   Copy URL → open in browser → add bot to your server\n\nStep 2 — n8n credential:\nSettings → Credentials → + Add → Discord\n  Bot Token: paste your bot token\n\nStep 3 — Discord Trigger in n8n:\n  Credential: your Discord bot\n  Channel: select or enter channel ID\n  Events: Message Created\n\nGet Channel ID:\n  Discord → User Settings → Advanced → Developer Mode: ON\n  Right-click channel → Copy Channel ID\n```\n\n---\n\n## Part 4 — WhatsApp: Complete Setup\n\n### WHY WhatsApp is Complex (and when to avoid it)\n\nUnlike Telegram (5 min setup) or Slack (20 min), WhatsApp requires:\n- A Meta Business account (verified business)\n- A phone number not already linked to WhatsApp personal\n- Approval from Meta (can take days)\n- Monthly costs after free tier\n\n**Recommendation:** Only use WhatsApp if your specific audience requires it. For personal or team automation, Telegram is simpler, free, and equally capable.\n\n### WhatsApp Options for n8n\n\n| Method | Difficulty | Cost | Reliability | Use Case |\n|---|---|---|---|---|\n| **Meta WhatsApp Business API** | Hard | Free (1000 msg/month), then paid | ✅ Official | Customer-facing bots |\n| **Waha (self-hosted)** | Medium | Free (open source) | ⚠️ Unofficial | Personal / small teams |\n| **WATI/Twilio/360Dialog** | Medium | Paid | ✅ Official | Business automation |\n\n### Option A — Meta WhatsApp Business API (Official)\n\n```\nStep 1 — Prerequisites:\n  - Meta Business account at: business.facebook.com\n  - Phone number not linked to WhatsApp personal\n  - Business verification may be required\n\nStep 2 — Create App:\n  1. developers.facebook.com → My Apps → Create App\n  2. App type: Business\n  3. Add Product: WhatsApp\n\nStep 3 — Set up phone number:\n  1. WhatsApp → Getting Started\n  2. Add phone number → verify\n  3. Get Phone Number ID and WhatsApp Business Account ID\n\nStep 4 — Get Access Token:\n  1. Temporary token (for testing): copy from Getting Started\n  2. Permanent token: System Users → generate token\n     Permissions: whatsapp_business_messaging\n\nStep 5 — Configure Webhook:\n  1. WhatsApp → Configuration → Webhook\n  2. Callback URL: https://your-ngrok-url.com/webhook/whatsapp\n  3. Verify Token: choose any string (e.g., \"my_verify_token\")\n  4. Subscribe to: messages\n\nStep 6 — In n8n:\n  Add Webhook trigger:\n    Path: whatsapp\n    Method: GET (for verification) AND POST (for messages)\n\n  Webhook verification handler (n8n Webhook node):\n  When Meta verifies: GET request with:\n    ?hub.mode=subscribe\n    ?hub.verify_token=my_verify_token\n    ?hub.challenge=SOME_NUMBER\n\n  Respond with the challenge:\n  If node: $json.query['hub.verify_token'] == 'my_verify_token'\n  → Respond to Webhook: {{ $json.query['hub.challenge'] }}\n\nStep 7 — Send message via HTTP Request:\n  Method: POST\n  URL: https://graph.facebook.com/v19.0/PHONE_NUMBER_ID/messages\n  Headers:\n    Authorization: Bearer YOUR_ACCESS_TOKEN\n    Content-Type: application/json\n  Body:\n  {\n    \"messaging_product\": \"whatsapp\",\n    \"to\": \"{{ $json.recipientPhone }}\",\n    \"type\": \"text\",\n    \"text\": { \"body\": \"{{ $json.message }}\" }\n  }\n\nOfficial docs: https://developers.facebook.com/docs/whatsapp/cloud-api/\n```\n\n### Option B — Waha (Self-hosted WhatsApp Gateway)\n\n**WHY Waha?** Free, open-source, runs in Docker, no Meta approval needed.\n\n**⚠️ Disclaimer:** Waha uses the WhatsApp web protocol. It's not officially supported by Meta. Use for personal projects only, not production customer communication.\n\n```\nStep 1 — Run Waha in Docker:\n```\n```yaml\n# Add to your docker-compose.yml:\n  waha:\n    image: devlikeapro/waha:latest\n    ports:\n      - \"3000:3000\"\n    volumes:\n      - waha_data:/app/.sessions\n\nvolumes:\n  waha_data:\n```\n```\nStep 2 — Start and connect WhatsApp:\n  docker compose up -d waha\n  Open: http://localhost:3000\n  Swagger UI shows all API endpoints\n\nStep 3 — Start a session:\n  POST http://localhost:3000/api/sessions/start\n  Body: { \"name\": \"default\" }\n\nStep 4 — Scan QR code:\n  GET http://localhost:3000/api/screenshot\n  Opens QR code → scan with WhatsApp on your phone\n\nStep 5 — Set Waha webhook:\n  POST http://localhost:3000/api/sessions/default/webhook\n  Body:\n  {\n    \"url\": \"https://your-ngrok-url.com/webhook/waha-messages\",\n    \"events\": [\"message\"]\n  }\n\nStep 6 — In n8n: Add Webhook trigger\n  Path: waha-messages\n  Method: POST\n  → Process incoming message: $json.payload.body\n\nStep 7 — Send message from n8n:\n  HTTP Request:\n  POST http://waha:3000/api/sendText\n  Body: { \"session\": \"default\", \"chatId\": \"{{ $json.phone }}@c.us\", \"text\": \"{{ $json.message }}\" }\n\nWaha docs: https://waha.devlike.pro\n```\n\n---\n\n## Part 5 — Messaging Platform
