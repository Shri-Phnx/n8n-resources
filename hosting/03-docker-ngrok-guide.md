# n8n Local Hosting with Docker Desktop + ngrok\n\n> Author: Shrinivas Ramaprasad | April 2026  \n> Source: https://docs.n8n.io/hosting/installation/docker/ and ngrok integration guide\n\n---\n\n## Overview — Why Docker + ngrok?\n\n| Without ngrok | With ngrok |\n|---|---|\n| n8n only accessible on `localhost:5678` | n8n accessible from the internet |\n| Webhooks from Stripe/GitHub/Telegram won't reach you | External services can reach your webhook |\n| Cannot test real integrations | Full real-world testing possible |\n| Cannot share n8n with teammates | Share public URL instantly |\n\n---\n\n## Part 1 — Install Docker Desktop\n\n### Windows\n```\n1. Go to: https://www.docker.com/products/docker-desktop\n2. Download for Windows → run Docker Desktop Installer.exe\n3. When prompted: Enable WSL2 integration\n   If WSL2 not installed: follow the link in the installer\n   https://aka.ms/wsl2install → takes 3-5 minutes\n4. Restart Windows when asked\n5. Docker Desktop opens automatically → wait for whale icon\n   in taskbar to become green = Docker is running\n6. Verify:\n   Open Command Prompt\n   > docker --version\n   > docker run hello-world\n```\n\n**⚠️ Windows issues:**\n| Problem | Fix |\n|---|---|\n| \"Hardware virtualization not enabled\" | Restart → BIOS → enable VT-x/AMD-V |\n| Docker won't start | Run as Administrator |\n| WSL2 kernel update required | Visit: aka.ms/wsl2kernel |\n\n### macOS\n```\n1. docker.com/products/docker-desktop → Download for Mac\n2. Choose: Apple Silicon (M1/M2/M3) OR Intel\n   Check: Apple menu → About This Mac → Chip\n3. Open .dmg → drag Docker to Applications\n4. Open Docker from Applications → allow helper installation\n5. Wait for menu bar whale icon to stop animating = running\n6. Verify: open Terminal → docker --version\n```\n\n**⚠️ macOS: \"App can't be opened\" error**\n```\nSystem Settings → Privacy & Security → Open Anyway\n```\n\n### Linux (Ubuntu/Debian)\n```bash\ncurl -fsSL https://get.docker.com | sh\nsudo usermod -aG docker $USER\nnewgrp docker\nsudo apt-get install -y docker-compose-plugin\ndocker --version && docker compose version\n```\n\n---\n\n## Part 2 — Run n8n in Docker\n\n### Step 1 — Create project folder\n```bash\n# Windows\nmkdir C:\\n8n && cd C:\\n8n\n\n# macOS / Linux\nmkdir ~/n8n && cd ~/n8n\n```\n\n### Step 2 — Create .env file\n```env\n# Generate encryption key: openssl rand -hex 32\nN8N_ENCRYPTION_KEY=paste-32-char-key-here\n\nN8N_BASIC_AUTH_ACTIVE=true\nN8N_BASIC_AUTH_USER=admin\nN8N_BASIC_AUTH_PASSWORD=YourStrongPassword!\n\nGENERIC_TIMEZONE=Asia/Kolkata\nTZ=Asia/Kolkata\n\n# Will be updated with ngrok URL in Step 4\nWEBHOOK_URL=http://localhost:5678\n```\n\n### Step 3 — Create docker-compose.yml\n```yaml\nversion: '3.8'\nservices:\n  n8n:\n    image: docker.n8n.io/n8nio/n8n:latest\n    # latest = always newest version\n    # For pinned version: docker.n8n.io/n8nio/n8n:1.45.0\n\n    container_name: n8n\n    restart: unless-stopped\n    # unless-stopped = restarts on crash/reboot but NOT on docker stop\n\n    ports:\n      - \"5678:5678\"\n    # Left = your machine port | Right = container port\n    # Change left to 8080 if 5678 is taken\n\n    env_file:\n      - .env\n    # Loads all variables from .env file\n\n    volumes:\n      - n8n_data:/home/node/.n8n\n    # CRITICAL: without this, ALL workflows lost on container restart\n    # n8n_data = Docker-managed named volume on your machine\n\nvolumes:\n  n8n_data:\n    # Stored at: /var/lib/docker/volumes/n8n_n8n_data (Linux)\n    # Managed by Docker Desktop VM (Windows/Mac)\n```\n\n### Step 4 — Start n8n\n```bash\n# Start (background mode)\ndocker compose up -d\n\n# Check status (look for \"Up\")\ndocker compose ps\n\n# View logs\ndocker compose logs n8n\n\n# Follow live logs\ndocker compose logs -f n8n\n```\n\nOpen browser: **http://localhost:5678**\n\n**⚠️ Common issues:**\n| Problem | Fix |\n|---|---|\n| Port 5678 already in use | Change left port: `\"5679:5678\"` |\n| Container keeps restarting | `docker compose logs n8n` → find error |\n| Cannot login | Check N8N_BASIC_AUTH_PASSWORD in .env |\n\n---\n\n## Part 3 — Set Up ngrok (Public HTTPS Tunnel)\n\n### Why ngrok?\nYour local n8n is at `localhost:5678` — not reachable from internet. ngrok creates a secure HTTPS tunnel that forwards `https://abc.ngrok-free.app` to your `localhost:5678`.\n\n### Step 1 — Install ngrok\n```\nWindows:\n  1. winget install ngrok  (or download from ngrok.com/download)\n  2. Open new Command Prompt → ngrok version\n\nmacOS:\n  brew install ngrok\n  OR: download .zip from ngrok.com/download → extract to /usr/local/bin\n\nLinux:\n  curl -s https://ngrok-agent.s3.amazonaws.com/ngrok.asc \\\n    | sudo tee /etc/apt/trusted.gpg.d/ngrok.asc > /dev/null\n  echo \"deb https://ngrok-agent.s3.amazonaws.com buster main\" \\\n    | sudo tee /etc/apt/sources.list.d/ngrok.list\n  sudo apt update && sudo apt install ngrok\n```\n\n### Step 2 — Create ngrok account and authenticate\n```\n1. Go to: https://ngrok.com → sign up (free)\n2. Dashboard → Your Authtoken → copy token\n3. Terminal:\n   ngrok config add-authtoken YOUR_AUTH_TOKEN_HERE\n   Saved to: ~/.config/ngrok/ngrok.yml\n```\n\n### Step 3 — Start ngrok tunnel\n```bash\nngrok http 5678\n```\n\n**You will see:**\n```\nForwarding  https://abc123.ngrok-free.app -> http://localhost:5678\nForwarding  http://abc123.ngrok-free.app  -> http://localhost:5678\n\nConnections  ttl  opn  rt1  rt5  p50  p90\n             0    0    0.00  0.00  -    -\n```\n\nYour n8n is now at: **https://abc123.ngrok-free.app**\n\n### Step 4 — Update WEBHOOK_URL in n8n\n\n```bash\n# Stop n8n\ndocker compose stop n8n\n\n# Update .env file:\nWEBHOOK_URL=https://abc123.ngrok-free.app\nN8N_EDITOR_BASE_URL=https://abc123.ngrok-free.app\n\n# Restart n8n\ndocker compose up -d n8n\n```\n\nNow open n8n via the ngrok URL: **https://abc123.ngrok-free.app**\n\nAll webhook URLs in n8n will now show the ngrok URL — external services can reach them.\n\n**⚠️ ngrok free plan limitation:**\nThe URL changes every time you restart ngrok. To keep a permanent URL:\n- ngrok paid plan ($8/month): fixed domain like `n8n.ngrok.app`\n- Free alternative: Cloudflare Tunnel (free, permanent domain)\n\n### ngrok Dashboard\nView all requests to your tunnel at: **http://127.0.0.1:4040**  \nGreat for debugging webhook payloads in real time.\n\n---\n\n## Part 4 — Updating n8n\n\n```bash\ncd ~/n8n\n\n# Backup first (always)\ndocker run --rm -v n8n_n8n_data:/source -v $(pwd)/backups:/backup \\\n  alpine tar czf /backup/n8n-$(date +%Y%m%d).tar.gz -C /source .\n\n# Pull new image\ndocker compose pull n8n\n\n# Recreate container (data preserved in volume)\ndocker compose up -d n8n\n\n# Verify new version\ndocker exec -it n8n n8n --version\n```\n\n**⚠️ Never run `docker volume rm n8n_data` — this deletes ALL your workflows.**\n\n---\n\n## Part 5 — Platform-Specific Notes\n\n### Windows: Access from other devices on LAN\n```\n1. Find your Windows IP: ipconfig → look for IPv4 Address\n   e.g., 192.168.1.105\n2. Access n8n from phone/another PC:\n   http://192.168.1.105:5678\n3. If blocked: Windows Defender Firewall → Allow port 5678\n```\n\n### macOS: Access from other devices on LAN\n```\nbash: ipconfig getifaddr en0  → your Mac's IP\nAccess: http://YOUR_MAC_IP:5678\n```\n\n### QNAP NAS: Run n8n via Container Station\n```\n1. Container Station → + Create\n2. Search: n8nio/n8n → select latest\n3. Port mapping: 5678 → 5678\n4. Volume: /share/n8n → /home/node/.n8n\n5. Environment variables: paste from .env\n6. Create and start\n7. Access: http://NAS_IP:5678\n```\n
