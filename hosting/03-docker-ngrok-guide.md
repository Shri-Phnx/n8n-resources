# n8n with Docker Desktop + ngrok — Complete Guide\n\n> **Author:** Shrinivas Ramaprasad | **Updated:** May 2026  \n> **Official docs:** https://docs.n8n.io/hosting/installation/docker/  \n> **Official n8n install page:** https://docs.n8n.io/hosting/installation/  \n> **ngrok docs:** https://ngrok.com/docs\n\n---\n\n## WHY — Why Docker? Why ngrok?\n\n### Why Docker instead of installing n8n directly?\n\n| Direct npm install | Docker |\n|---|---|\n| Tied to your machine's Node.js version | Runs in isolated container |\n| Breaks when Node.js updates | Never breaks — consistent environment |\n| Hard to move to another machine | Move by copying 2 files |\n| No automatic restart on crash | `restart: unless-stopped` handles it |\n| Hard to add PostgreSQL/Ollama | Add services in one docker-compose.yml |\n| Difficult cleanup | `docker compose down` = clean remove |\n\n**Bottom line:** Docker is the recommended way to run n8n. It's what n8n.io themselves use and document first.\n\n**Reference:** https://docs.n8n.io/hosting/installation/docker/\n\n### Why ngrok?\n\nWhen you run n8n on your laptop, it's only accessible at `http://localhost:5678`. That URL works only on YOUR device. External services like Telegram, Stripe, GitHub, and Slack need to reach your n8n to send webhook events — they can't reach `localhost`.\n\nngrok creates a **secure public HTTPS tunnel**:\n\n```\nInternet Request:\nTelegram / Stripe / GitHub\n        ↓\nhttps://abc123.ngrok-free.app   ← Public URL anyone can reach\n        ↓  (encrypted tunnel)\nngrok agent (running on your machine)\n        ↓\nhttp://localhost:5678            ← Your local n8n\n        ↓\nn8n processes the webhook\n```\n\n**Without ngrok:** Webhook triggers (Telegram, Stripe, GitHub) won't work on a laptop.  \n**With ngrok:** Your laptop works exactly like a cloud server for webhook testing.\n\n---\n\n## WHERE — Where Does Everything Run?\n\n```\nYour Laptop / PC\n├── Docker Desktop (manages containers)\n│   ├── n8n container (port 5678)\n│   ├── [optional] PostgreSQL container\n│   └── [optional] Ollama container (AI models)\n└── ngrok agent (separate process)\n    └── Tunnel: https://abc.ngrok-free.app → localhost:5678\n\nExternal Services\n├── Telegram Bot API → sends messages to ngrok URL\n├── Stripe → sends payments to ngrok URL\n├── GitHub → sends push events to ngrok URL\n└── Any webhook source → reaches n8n via ngrok\n```\n\n---\n\n## Part 1 — Install Docker Desktop\n\n### Windows — Step by Step\n\n**What is WSL2?** Windows Subsystem for Linux 2. Delete container = lose everything.\n- **env_file:** Separates secrets from config. Never hardcode passwords in docker-compose.yml.\n- **restart policy:** Ensures n8n auto-restarts if it crashes or your machine reboots.\n- **GENERIC_TIMEZONE:** Without this, Schedule Triggers fire in UTC — not your local time.\n\n### Step 1 — Create your n8n project folder\n\n```bash\n# Windows (Command Prompt)\nmkdir C:\\n8n\ncd C:\\n8n\n\n# macOS / Linux\nmkdir ~/n8n\ncd ~/n8n\n```\n\n### Step 2 — Generate your encryption key\n\n**WHY:** n8n encrypts all credentials (API keys, passwords) using this key. If you lose it or change it, ALL your saved credentials become permanently unreadable.\n\n```bash\n# macOS / Linux — run this command, copy the output\nopenssl rand -hex 32\n# Output example: a8f3b2c1d4e5f6a7b8c9d0e1f2a3b4c5d6e7f8a9b0c1d2e3f4a5b6c7d8e9f0a1\n\n# Windows PowerShell\n[System.Convert]::ToBase64String([System.Security.Cryptography.RandomNumberGenerator]::GetBytes(32))\n```\n\n**Save this key in your password manager NOW before proceeding.**\n\n### Step 3 — Create .env file\n\nCreate a file named `.env` (no extension) in `C:\\n8n` or `~/n8n`:\n\n```env\n# ==============================================\n# n8n Configuration File\n# NEVER commit this file to Git\n# ==============================================\n\n# Encryption key — CRITICAL. Never change after first run.\n# Generate with: openssl rand -hex 32\nN8N_ENCRYPTION_KEY=paste-your-generated-key-here\n\n# Login credentials for n8n UI\nN8N_BASIC_AUTH_ACTIVE=true\nN8N_BASIC_AUTH_USER=admin\nN8N_BASIC_AUTH_PASSWORD=YourStrongPassword!\n\n# Timezone — IMPORTANT for Schedule Triggers\n# Find yours at: https://en.wikipedia.org/wiki/List_of_tz_database_time_zones\nGENERIC_TIMEZONE=Asia/Kolkata\nTZ=Asia/Kolkata\n\n# Webhook URL — update this when you get your ngrok URL\n# Format: https://your-public-url.com (no trailing slash)\nWEBHOOK_URL=http://localhost:5678\n\n# Optional: where n8n UI is accessible\nN8N_EDITOR_BASE_URL=http://localhost:5678\n```\n\n### Step 4 — Create docker-compose.yml\n\nCreate `docker-compose.yml` in the same folder:\n\n```yaml\n# n8n Docker Compose Configuration\n# Reference: https://docs.n8n.io/hosting/installation/docker/\n\nversion: '3.8'\n\nservices:\n  n8n:\n    # Official n8n Docker image from n8n's own registry\n    # Registry: https://hub.docker.com/r/n8nio/n8n\n    image: docker.n8n.io/n8nio/n8n:latest\n    # Use a pinned version in production:\n    # image: docker.n8n.io/n8nio/n8n:1.45.0\n\n    container_name: n8n\n    # Gives the container a fixed name so you can reference it\n    # docker logs n8n  /  docker exec -it n8n sh\n\n    restart: unless-stopped\n    # Options:\n    # 'no'             = never restart automatically\n    # 'always'         = always restart (even after docker stop)\n    # 'unless-stopped' = restart on crash/reboot, NOT on docker stop ← RECOMMENDED\n    # 'on-failure'     = only restart if exits with error\n\n    ports:\n      - \"5678:5678\"\n    # Format: \"HOST_PORT:CONTAINER_PORT\"\n    # 5678:5678 means:\n    #   → You access n8n at: http://localhost:5678\n    #   → Inside container n8n runs on: 5678\n    # Change host port if 5678 is taken: \"8080:5678\"\n\n    env_file:\n      - .env\n    # Loads all KEY=VALUE pairs from .env file\n    # Keeps secrets out of docker-compose.yml (which you might share)\n\n    volumes:\n      - n8n_data:/home/node/.n8n\n    # /home/node/.n8n is where n8n stores EVERYTHING:\n    #   - database.sqlite (all workflows, credentials, executions)\n    #   - encryption key file\n    #   - config files\n    # n8n_data is a Docker-managed named volume on your machine\n    # ⚠️ Without this line: deleting/recreating the container = losing ALL workflows\n\nvolumes:\n  n8n_data:\n  # Docker stores this at:\n  # Linux: /var/lib/docker/volumes/n8n_n8n_data/_data\n  # Windows/Mac: Inside Docker Desktop's VM (not directly visible)\n  # To inspect: docker volume inspect n8n_n8n_data\n```\n\n### Step 5 — Start n8n\n\n```bash\n# Navigate to your n8n folder\ncd ~/n8n     # or cd C:\\n8n on Windows\n\n# Start n8n in background (detached mode)\ndocker compose up -d\n\n# Check it started successfully\ndocker compose ps\n# Expected output:\n# NAME    IMAGE              COMMAND    STATUS    PORTS\n# n8n     .../n8n:latest     ...        Up        0.0.0.0:5678->5678/tcp\n\n# View startup logs (useful for debugging)\ndocker compose logs n8n\n\n# Follow live logs (press Ctrl+C to stop following)\ndocker compose logs -f n8n\n# Look for: \"n8n ready on 0.0.0.0, port 5678\"\n```\n\nOpen browser: **http://localhost:5678**\n\nFirst run: n8n asks you to create an owner account (name, email, password). This is a local account — nothing is sent to n8n servers.\n\n**⚠️ Startup troubleshooting:**\n| Problem | What to check | Fix |\n|---|---|---|\n| `Error: bind: address already in use` | Port 5678 taken | Change to `\"5679:5678\"` in compose file |\n| Container exits immediately | Config error | `docker compose logs n8n` → find the error line |\n| Can't login / white screen | Wrong password | Check N8N_BASIC_AUTH_PASSWORD in .env |\n| Credentials show \"cannot decrypt\" | Encryption key changed | Restore original N8N_ENCRYPTION_KEY |\n\n---\n\n## Part 3 — ngrok: Expose n8n to the Internet\n\n### WHEN do you need ngrok?\n\nYou need ngrok when **any external service needs to reach your n8n**:\n\n| Trigger / Node | Needs Public URL? | Works on localhost? |\n|---|---|---|\n| Manual Trigger | ❌ No | ✅ Yes |\n| Schedule Trigger | ❌ No | ✅ Yes |\n| Webhook Trigger | ✅ YES | ❌ No |\n| Telegram Trigger (webhook mode) | ✅ YES | ❌ No |\n| Slack Trigger | ✅ YES | ❌ No |\n| GitHub Trigger | ✅ YES | ❌ No |\n| Stripe Trigger | ✅ YES | ❌ No |\n| Gmail Trigger (polling mode) | ❌ No | ✅ Yes |\n| n8n Form Trigger | ✅ YES (for external users) | ✅ Locally only |\n\n**Simple rule:** Any trigger that says \"webhook\" needs a public URL = needs ngrok.\n\n### Step 1 — Create ngrok account\n\n```\n1.
