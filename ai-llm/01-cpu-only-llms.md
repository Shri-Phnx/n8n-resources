# CPU-Only LLMs — Run AI Without a GPU\n\n> **Author:** Shrinivas Ramaprasad | **Updated:** May 2026\n> **Reference:** https://localaimaster.com/blog/ollama-system-requirements\n\n---\n\n## Why This Matters for n8n\n\nMost guides assume you have an NVIDIA GPU. If you're running n8n on a laptop, a NAS, a budget VPS, or a Mac — you may have no dedicated GPU at all. The good news: **you can run capable LLMs on CPU only**. This guide tells you exactly which models work, how fast they'll be, how to configure them in n8n, and what to do when context limits hit.\n\n---\n\n## The Key Facts\n\n- **CPU inference works.** Ollama runs on CPU with no GPU present.\n- **Speed is slower.** Expect 5–15 tokens/second on a modern CPU vs 40–80+ on a GPU.\n- **RAM is the constraint**, not VRAM. You need ~0.6 GB RAM per billion parameters at Q4 quantization.\n- **Quantization is your friend.** A 7B model at Q4 (4-bit) uses ~4.5 GB RAM instead of 14 GB at FP16.\n- **Apple Silicon (M1/M2/M3/M4) is the exception** — unified memory means these chips run LLMs almost as fast as a mid-range GPU.\n\n---\n\n## Hardware Requirements by Model Size\n\n| Model Size | Min RAM (CPU) | Comfortable RAM | Tokens/sec (CPU) | Notes |\n|---|---|---|---|---|\n| **0.5B–1B params** | 2 GB | 4 GB | 20–40 t/s | Very fast on CPU. Limited capability. |\n| **3B–4B params** | 4 GB | 6 GB | 10–20 t/s | Sweet spot for CPU-only. Good quality. |\n| **7B–8B params** | 8 GB | 12 GB | 5–12 t/s | Usable. Slow for long responses. |\n| **13B–14B params** | 12 GB | 16 GB | 3–6 t/s | Slow on most CPUs. |\n| **32B+ params** | 24 GB | 32 GB | 1–3 t/s | Painful on CPU. Not recommended. |\n\n> **Rule of thumb:** If you have 16 GB RAM → run up to 8B models. 8 GB RAM → stick to 3B–4B models.\n\n---\n\n## Recommended Models for CPU-Only Use\n\n### Tier 1: Best for 4–6 CPU models tend to be smaller and have shorter context windows. When your workflow sends too much text, the model either truncates it or throws an error.\n\n| Model | Context Window | Max Input (practical) |\n|---|---|---|\n| Phi-3 Mini | 4,096 tokens | ~2,500 words input |\n| Llama3.2:3B | 128,000 tokens | Very long |\n| Gemma3:1B | 8,192 tokens | ~5,000 words |\n| Mistral:7B | 32,768 tokens | ~20,000 words |\n| Phi-4 Mini | 16,384 tokens | ~10,000 words |\n\n> ~1 token ≈ 0.75 words in English.\n\n### Solution 1: Text Splitter + Map Reduce (Best for Documents)\n\n```\nInput Document (long)\n    ↓\nRecursive Character Text Splitter\n    Chunk size: 1000 tokens\n    Overlap: 100 tokens\n    ↓\nLoop Over Items\n    ↓\n  Basic LLM Chain (summarise/extract each chunk)\n  Model: phi3:mini (fits in 4K context)\n    ↓\nMerge all chunk results\n    ↓\nBasic LLM Chain (final summary of summaries)\n```\n\n### Solution 2: Multi-Model Fallback Chain\n\n```\nInput Text\n    ↓\nCode Node: check token count\n  if (text.length < 2000) → use phi3:mini (fast, CPU)\n  else if (text.length < 8000) → use llama3.2:3b (slightly slower)\n  else → use groq API (free cloud, no context limit problem)\n    ↓\nIf Node: which model?\n    ├── Short → Ollama phi3:mini\n    ├── Medium → Ollama llama3.2:3b\n    └── Long → Groq Cloud API (backup)\n```\n\n### Solution 3: Summarise Before Processing\n\n```\nLong Input\n    ↓\nSummarization Chain\n    Model: llama3.2:3b (128K context)\n    Mode: Map Reduce\n    ↓\nShort Summary (~500 words)\n    ↓\nMain Processing Chain\n    Model: phi3:mini (fine for 500 words)\n```\n\n### Solution 4: Context Trimming in Code Node\n\n```javascript\n// Code Node — trim input to fit model context\nconst text = $json.inputText;\nconst MAX_CHARS = 3000; // ~2000 tokens for phi3:mini\n\nconst trimmed = text.length > MAX_CHARS\n  ? text.substring(0, MAX_CHARS) + '... [truncated]'\n  : text;\n\nreturn [{ json: { text: trimmed, was_truncated: text.length > MAX_CHARS } }];\n```\n\n---\n\n## Multi-Provider Fallback Strategy for n8n\n\nWhen a free API provider hits its rate limit or context limit, automatically fall back to the next one:\n\n```\nInput\n  ↓\nIf: text < 3000 chars AND groq_calls_today < 14000\n  ↓ YES\n  Groq API (fastest, free)\n  ↓ NO\nIf: text < 6000 chars\n  ↓ YES\n  Ollama Phi-3 Mini (local, free)\n  ↓ NO\nIf: text < 16000 chars\n  ↓ YES\n  Ollama Llama3.2:3B (local, 128K context)\n  ↓ NO\nGoogle Gemini API (free tier, 1M+ context)\n```\n\n### Rate limits reference for fallback logic\n\n| Provider | Daily Limit | RPM | Context |\n|---|---|---|---|\n| **Groq** (free) | 14,400 req | 30 | 32K–128K |\n| **OpenRouter** free models | 50 req/day | 20 | varies |\n| **Google Gemini** (free) | 1,500 req | 15 | 1M |\n| **Mistral** (free) | 500K tokens/month | 1 RPS | 128K |\n| **Ollama** (local) | Unlimited | Hardware | Model-dependent |\n\n---\n\n## LLM Tools — No GPU Required\n\n| Tool | What it is | Platform | URL |\n|---|---|---|---|\n| **Ollama** | CLI runner, REST API server | Win/Mac/Linux | https://ollama.com |\n| **LM Studio** | GUI with built-in chat + API server | Win/Mac/Linux | https://lmstudio.ai |\n| **GPT4All** | One-click GUI, model browser | Win/Mac/Linux | https://gpt4all.io |\n| **Jan** | Open-source ChatGPT-style desktop app | Win/Mac/Linux | https://jan.ai |\n| **Open WebUI** | Browser-based chat UI for Ollama | Any (Docker) | https://openwebui.com |\n\n**Recommendation:** Ollama for n8n integration (REST API, easiest).
